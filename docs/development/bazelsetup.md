# Bazel Integration Guide for KubeAir

## Overview

KubeAir now includes **Bazel 9.x with Modules** (no WORKSPACE) for building Rust code with automatic BUILD file generation via **Gazelle**.

## Files Added

### Core Configuration

| File | Purpose |
|------|---------|
| `MODULE.bazel` | Bazel module declaration; defines Rust toolchain, gazelle, and crate_universe |
| `.bazelversion` | Pins Bazel version (9.1.1) |
| `.bazelrc` | Build configuration (test output, etc.) |
| `.bazelignore` | Directories to ignore during analysis |
| `BUILD.bazel` | Root BUILD with Gazelle target |
| `crates/BUILD.bazel` | Crates directory marker |

### Documentation

| File | Purpose |
|------|---------|
| `README.md` | Quick start guide |
| `docs/development/bazelsetup.md` | Bazel setup guide |

## Getting Started

### Installation

If Bazelisk is not installed:

```bash
brew install bazelisk
```

Bazelisk automatically downloads the correct Bazel version from `.bazelversion`.

### First Time Setup

```bash
# 1. Fetch all dependencies
bazel fetch //...

# 2. Run Gazelle to generate BUILD files
bazel run //:gazelle -- -mode=fix

# 3. Query available targets
bazel query //...
```

## Building and Testing

### Build a Single Crate

    bazel build //crates/kubelet-core:lib

### Run Tests

    # Run all tests
    bazel test //...

    # Run a specific suite
    bazel test //tests/integration:pod_lifecycle_test

### Run the Binary

Once BUILD files are generated, the root binary can be run:

    bazel run //:kubelet

## Dependency Management

Dependencies are managed via **crate_universe**, which reads `Cargo.toml` and `Cargo.lock`:

1. If you add a new dependency to `Cargo.toml`, run `cargo update` to lock versions
2. Re-run Gazelle: `bazel run //:gazelle -- -mode=fix`
3. Bazel will automatically discover and use new crates

### Manual Dependency Resolution

If crate_universe has issues (e.g., incompatible versions), you can:

1. Check available crates: `bazel query '@crates//:*'`
2. Manually add deps in BUILD files: `"@crates//:package_name"`

## Gazelle Configuration

Gazelle is configured in `BUILD.bazel` with the prefix `github.com/BenCoxfordDev/kubeair`. To customize:

- Edit the `gazelle()` rule in the root `BUILD.bazel`
- Add `# gazelle:directive value` comments in any BUILD file

### Useful Gazelle Commands

```bash
# Preview changes without applying
bazel run //:gazelle -- -mode=print | head -50

# Update BUILD files
bazel run //:gazelle -- -mode=fix

# Verbose output
bazel run //:gazelle -- -v
```

## Troubleshooting

### "no such target" errors

Ensure Gazelle has generated BUILD files:

```bash
bazel run //:gazelle -- -mode=fix
```

### Build failures with missing crates

1. Verify Cargo.lock is up to date: `cargo update`
2. Clear Bazel cache: `bazel clean --expunge`
3. Re-fetch: `bazel fetch //...`

### Bazel version mismatch

If you get "Bazel version X expected", either:

- Update `.bazelversion` to match your installed Bazel
- Or install Bazelisk, which respects `.bazelversion`

### Proto file issues

Proto BUILD files are auto-generated by Gazelle. If you see Go-related rules, either:

- Delete them and re-run Gazelle in Rust-only mode
- Or add `# gazelle:build_file_name BUILD.bazel` directives to configure language-specific generation

## Integration with Cargo

You can use Bazel and Cargo simultaneously:

- **Cargo**: `cargo build`, `cargo test` (traditional Rust workflow)
- **Bazel**: `bazel build //...`, `bazel test //...` (reproducible, cacheable builds)

Both share the same source tree, so changes are immediately visible to each tool.

## Performance Tips

- Use `bazel clean` to clear stale outputs: `bazel clean`
- Speed up rebuilds with `--spawn_strategy=local`: `bazel build --spawn_strategy=local //...`
- Enable remote caching for CI/CD (see Bazel docs)
